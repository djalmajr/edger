# Code of conduct

EdgeR contributors and maintainers are expected to keep project spaces
professional, respectful and focused on improving the software.

## Expected behavior

- Give actionable, technically grounded feedback.
- Respect different backgrounds, experience levels and viewpoints.
- Assume good faith while addressing observable behavior directly.
- Keep private disclosures, personal information and credentials confidential.
- Accept maintainer decisions about project scope and product boundaries.

## Unacceptable behavior

Harassment, threats, discriminatory language, personal attacks, deliberate
disruption, doxxing and publishing another person's private information are not
accepted in project spaces.

## Enforcement

Report conduct concerns privately to the repository maintainer. Maintainers may
edit or remove contributions and temporarily or permanently restrict
participation when behavior is incompatible with this policy. Reports will be
handled as confidentially as practical.
