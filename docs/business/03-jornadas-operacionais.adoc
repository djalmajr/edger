=== Jornadas operacionais

==== Subir o runtime local

1. Garanta que Rust/Cargo e Deno estão disponíveis.
2. Inicie o binário Rust com `ROOT_API_KEY` local:
+
[source,bash]
----
ROOT_API_KEY=test-root PORT=19080 RUNTIME_WORKER_DIRS=workers/examples \
  EDGER_CORE_WORKER_DIR=workers/core \
  EDGER_CORE_WORKER_OVERLAY_DIR=.edger/core-worker-overlays \
  cargo run -p edger-orchestrator --bin edger
----
3. Valide readiness:
+
[source,bash]
----
curl http://127.0.0.1:19080/ready
----

==== Adicionar um worker JS/TS

1. Para exemplos versionados no repositório, crie um diretório em
   `workers/examples/<name>/`. Em produção, use um diretório de usuário
   configurado em `RUNTIME_WORKER_DIRS`.
2. Adicione `index.ts`, `index.js` ou `index.mjs`.
3. Use uma forma suportada:
+
[source,ts]
----
Deno.serve((req) => new Response(`path=${new URL(req.url).pathname}`));
----
+
ou:
+
[source,ts]
----
export default {
  fetch(req: Request) {
    return new Response(req.method);
  },
};
----
4. Opcionalmente adicione `manifest.yaml` com `name`, `version`, `timeout` e
   `kind`.
5. Chame `/<name>` com `Authorization: Bearer <ROOT_API_KEY>`.

==== Adicionar um worker Wasm

1. Crie `workers/examples/<name>/manifest.yaml` com `kind: wasm` e `entrypoint`
   para uma fixture local, ou use um root de usuário configurado em produção.
2. Adicione `index.wasm` ou `index.wat`.
3. Exporte `memory`, `edger_alloc(len)` e `edger_handle(ptr, len)` conforme
   `planning/edger/docs/wasm-abi.md`.
4. Rode o servidor e chame `/<name>`.

==== Criar uma nova extensão

1. Crie um crate `crates/edger-ext-<nome>`.
2. Dependa apenas de `edger-core`.
3. Escolha um modo: `Middleware`, `AuthProvider` ou `WorkerHandler`.
4. Exporte uma factory clara, como `middleware()`.
5. Registre explicitamente no bin `edger`.
6. Rode `cargo test -p edger-ext-<nome>` e o gate Rust do workspace.

==== Validar compatibilidade Buntime

Use `planning/edger/docs/compat-matrix.md` como trilha de compatibilidade. O
estado atual já cobre fetch JS/TS, `Deno.serve`, default fetch, path relativo,
`x-base`, alguns streams com fallback de primeiro chunk e Wasm v1. Shell, cron,
routes table, StaticSpa production e observabilidade ainda dependem da Fase 7.
