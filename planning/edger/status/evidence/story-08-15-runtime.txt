# Story 08.15 runtime evidence — gateway redirect rules

Date: 2026-06-29

Scope:
- `edger-ext-gateway` now supports ordered redirect rules by path prefix.
- Redirect responses short-circuit with `Location`, status `308` by default,
  and status limited through the public builder to `301`, `302`, `307` or `308`.
- Redirect matching is segment-aware: `/api` matches `/api` and `/api/users`,
  not `/apiary`.
- Redirects preserve path suffix and query string.
- CORS preflight remains a 204 response and wins over redirect rules.
- `SerializedRequest.uri` now preserves `path?query`, and the orchestrator
  pipeline preserves query after rewriting the worker-relative path.
- No upstream HTTP proxy, regex rewrite, cache, rate-limit persistence,
  dynamic admin API or durable redirect storage in this slice.
