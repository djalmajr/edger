
left unchanged.

## Behavior covered

- Requests without `x-request-id` receive a generated UUID before worker
  dispatch.
- The generated request id is echoed in the response and observed by the worker
  through both the serialized request field and request header.
- Worker dispatch logs include `request_id`, `worker_name`, version and
  namespace.
- Dispatch logs do not include Authorization headers, the root key or request
  body content.
- `/metrics` exposes `edger_http_requests_total` and
  `edger_http_request_duration_ms_last`.
- `/metrics` still exposes pool/cache/spawn/request duration, ephemeral and cron
  counters without secrets.
- `TracingInitConfig` prefers `EDGER_LOG`, falls back to `RUST_LOG`, has a safe
  default filter and parses `OTEL_EXPORTER_OTLP_ENDPOINT` plus
  `OTEL_TRACES_SAMPLER`.
- Startup with OTEL env configured is non-fatal; actual OTLP export remains a
  follow-up because the exporter layer is not linked in this build.
- `WorkerPool::fetch_worker` and isolate execution dispatch are instrumented
  with spans carrying request/worker/kind context.
