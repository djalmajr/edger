
Result:

```text
running 8 tests
test root_lists_workers_with_operational_metadata ... ok
test admin_workers_requires_root_and_does_not_fall_through_to_api_stub ... ok
test worker_mutation_is_protected_before_not_implemented_response ... ok
test root_lists_key_metadata_without_raw_secret ... ok
test root_lists_registered_extensions_and_auth_provider ... ok
test root_creates_key_once_without_leaking_raw_secret_in_lists ... ok

test result: ok. 8 passed; 0 failed
```

Covered behavior:

- `POST /api/admin/keys` with root credentials returns `201`, safe metadata and one-time `rawKey`.
- Created key can authenticate against `/api/admin/session`.
- `GET /api/admin/keys` does not include the raw generated key.
- Non-root keys cannot create or revoke keys.
- Browser-originated mismatched origin is denied by CSRF guard before create.
- `POST /api/admin/keys/{id}/revoke` invalidates the key.
- Repeating revoke is idempotent and returns `revoked=false`.
