# Matriz de paridade de valor Buntime -> edger

**Origin:** `planning/edger/epics/08-valor-buntime/00-overview.md`

Esta matriz define “mesmo valor” por capacidade observável. Ela não exige copiar
implementação, nomes internos ou decisões específicas de Bun.

## Escala

| Campo | Valores |
|---|---|
| Prioridade | `must`, `should`, `later` |
| Status | `tested`, `partial`, `planned`, `gap`, `deferred` |
| Evidência | Teste automatizado, runbook, checkpoint, validação Browser ou decisão de escopo |

## Regras de atualização
- `tested`: há teste, runbook ou evidência manual versionada no repo.
- `partial`: o valor existe em parte, com lacuna explícita.
- `planned`: story definida, sem evidência executada.
- `gap`: sem contrato aceito ainda.
- `deferred`: decisão consciente fora do escopo desta fase.

## Relação com compatibilidade técnica
