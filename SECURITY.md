# Security policy

## Supported versions

EdgeR is still preparing its first public release. Until `v0.2.0` is published,
security fixes target the latest commit on `main`. The supported-version table
will be updated with each release line.

## Reporting a vulnerability

Do not open a public issue for an unpatched vulnerability. Use GitHub's
**Security > Report a vulnerability** flow to create a private security
advisory:

<https://github.com/djalmajr/edger/security/advisories/new>

Include the affected version, reproduction steps, impact and any suggested
mitigation. Avoid including production secrets or personal data.

The maintainer will acknowledge the report, assess severity and coordinate a
disclosure timeline. Security invariants required to run EdgeR safely belong to
the Community runtime and are not reserved for a commercial edition.

Private vulnerability reporting must be enabled in the repository security
settings before the repository is made public.
